Risposte domande Guarnieri

1) Che cos'è github e quali sono le sue caratteristiche?

Github è un software per il controllo delle versioni che nel corso degli ultimi anni è diventato molto popolare per le varie funzionalità che offre.
Infatti grazie a questo software è possibile lavorare ad un progetto e condividerlo con una cerchia definita di utenti (sfruttando la versione a pagamento) oppure chiedere il parere di tutta la community caricandolo pubblicamente (sfruttando la versione gratuita). La forza di github sta nel fatto che una copia del progetto può essere scaricata localmente da tutti quelli che ne hanno il permesso e possono effettuare modifiche direttamente sul codice originale o creare nuove versioni del codice a cui i proprietari del progetto possono dare una valutazione e decidere se includerlo o meno nel progetto principale. Una delle funzionalità più usate è il sistema di scambio di messaggi per la community, funzionalità che ha permesso la crescita del software in termini di popolarità e fa si che github sia uno dei migliori software di controllo delle versioni.

2) Come si crea un repository privato?

Per creare una repository è necessario accedere con il proprio acount github alla pagina ufficiale del sito. Una volta definito il nome ed una descrizione (opzionale) è possibile impostare dei parametri di personalizzazione della repository. Tra questi parametri c'è la possibilità di definire la repository come pubblica o privata. Sarà necessario cliccare vicino alla scelta per la repository privata e si verrà poi indirizzati ad una pagina che definirà il pagamento da effettuare per poter usufluire di questa funzionalità. Infatti github mette a disposizione una versione gratuita solo a chi crea tutte repository pubbliche e quindi condivise con tutta la community. Invece impostando il parametro privato sarà possibile nascondere alla community il proprio progetto e si potranno definire in un secondo momento i propri collaboratori. 

3) Cosa si intende per fare un branch?

Questa funzionalità è necessaria se si vuole creare un'altra versione del codice del progetto o semplicemente se si vuole collaborare ad un progetto di cui non conosciamo il proprietario ma vogliamo offrire un contributo.
Il branch è una funzionalità che crea una copia del progetto su cui il programmatore o l'utente può lavorare in completa autonomia senza avere il timore di modificare il codice primario presente su github. Una volta creato un brunch su github sarà possibile notare che è stata creata una nuova versione del progetto sulla quale si possono fare tutte le modifiche che si vogliono. Una volta concluse tutte le modifiche, si farà una richesta di commit ovvero si manderà una richiesta al proprietario del progetto invitandolo a prendere visione del lavoro svolto. Attraverso un sistema di post sarà possibile tra i due scambiarsi messaggi per eventuali correzioni. In caso in cui il proprietario accetti le modifiche includerà di conseguenza il branch nel progetto originale così tutti avranno, attraverso un Pull, tutte le nuove modifiche aggiornate.

4) Che cos'è e a cosa serve una issue?

Un'issue è un'assegnazione di una determinata task ad uno specifico collaboratore creata e gestita dal proprietario del progetto. Serve a definire un piano di lavoro ben definito per non rischiare di avere delle incongruenze nel codice nel caso in cui tutti facciano modifiche contemporaneamente e quindi modificando erroneamente il codice.
